函数中this的指向

发表于 2018-08-04

看似简单　并不简单

先看看下面这个题目

运行下面代码
会输出什么结果

先留下你得到的结果吧
总共有3个输出结果

第一个this

按照普通的思路去分析
function中this的指向是
谁调用，指向谁

那么上面的代码里面
首先是obj调用自身的method
传入 function fn 和 number 666

那么method指向的funcion被调用
因为是被obj调用
所以method指向的function中this的指向是obj

也就是
this ==> obj
this.length ==> obj.length

那么congosle.log的输出结果就是obj.lenth ==> 4

第二个this

接着是fn在method指向的function中被调用

这里并没有一个被声明的对象去调用fn

虽然fn是在obj的method所指向的function中被调用
但是并不是obj调用fn

那么这里将会由window去调用fn
谁调用，指向谁
显然，这里的运行结果是
this ==> window
this.length ==> window.length

那么congosle.log的输出结果就是window.lenth ==> 5

第三个this

最后是arguments[0]被调用
关于argumens
刚开始是这样想的
arguments[0]也就是fn
argumtns[0] ==> fn
也就是调用fn
那么输出结果跟第二个this一样，是5

并不是这么简单

第三个this
也就是这个arguments中的fn中的this
既不是指向obj
也不是指向window
更不可能是传入的第二个参数666

那么这里的this到底指向谁
等等……
好像错过了什么重要的东西

第二个参数 ==> 两个参数
两个 ==> 2

难道说这里的fn中this指向arguments
那么也就是说

this ==> arguments
this.length ==> arguments.this
那么congosle.log的输出结果就是arguments.lenth ==> 2

总结

在寻找第三个this指向的时候
有一个点被我忽略了
arguments是一个伪数组（对象）
而不是一个数组

接下来
手写一个伪数组
那么一切将会变得更清晰明了

这时候 obj[3] ( ) 相当于是 obj.3 ( )
但是后者的写法是错误的
所以只能用第一种写法
也就是说
是obj调用了fn
fn的this指向arguments

arguments


函数中的arguments是一个伪数组，它包含着所有被传入的实参
在argumens中的形参以值(value)的形式储存，每一个形参都有其对应的一个键(key)
也就是说是以键值对(key: value)的方式储存的
另外还有一个特别的键——length，它的值是arguments中对应的函数所接收实参数量
也就是函数接受到多少个参数，length的值就是多少
